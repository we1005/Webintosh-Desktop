# 机器学习套件使用指南

Webintosh 内置了 4 个面向机器学习 / 深度学习学习与演示的应用,全部**在浏览器里运行,无需安装任何本地环境**。本文档说明每个应用怎么用、背后是什么原理、能做什么、不能做什么。

| 应用 | 底层项目 | 一句话定位 |
|---|---|---|
| 🐍 Python 实验室 | [Pyodide](https://github.com/pyodide/pyodide) | 浏览器里的 Python + 科学计算全家桶 |
| 🤖 AI 工具箱 | [transformers.js](https://github.com/huggingface/transformers.js) | 本地跑预训练深度学习模型(推理) |
| 📓 JupyterLite | [jupyterlite](https://github.com/jupyterlite/jupyterlite) | 零后端 Jupyter Notebook |
| 🕹️ 复古游戏厅 | [EmulatorJS](https://github.com/EmulatorJS/EmulatorJS) | 浏览器复古主机模拟器(娱乐) |

**共同前提**:这 4 个应用的运行时与模型都从 CDN 按需加载,**需要联网**;首次加载后浏览器会缓存,二次使用快很多。全部从**启动台**启动(Dock → 启动台 → 对应图标),关闭窗口后图标自动从 Dock 消失。

## 一、Python 实验室

### 怎么用(流程)

1. 启动台 → **Python 实验室**,等待状态栏变绿「Python 3.12 就绪」(首次约 10–30s,加载 ~10MB 运行时)
2. 左侧写 Python 代码 → 点 **▶ 运行** 或按 **⌘↵**
3. `print` 输出显示在右侧;**matplotlib 图表**调用 `show()` 后显示在右下
4. 顶部下拉菜单有 6 个**内置 ML 教学示例**,选中即填入代码:
   - 入门:NumPy 矩阵运算(矩阵乘法、特征值、逆)
   - **线性回归**:最小二乘闭式解 + 散点/拟合线可视化
   - **KMeans 聚类**:sklearn + 彩色散点图(首次会自动下载 sklearn ~25MB)
   - **决策边界**:逻辑回归 vs RBF 核 SVM 在月牙数据上的对比
   - pandas 数据分析:分组统计 + 柱状图
   - **梯度下降**:在碗状函数等高线上画出一步步下降的路径

学习建议:改示例里的参数再跑——比如梯度下降的学习率 `lr` 改成 0.3(震荡)或 0.01(慢),立刻看到优化行为变化;KMeans 的 `n_clusters` 改错看看会怎样。

### 原理

- **Pyodide = CPython 解释器编译成 WebAssembly**(经 Emscripten),你的代码由真正的 CPython 3.12 在浏览器沙箱里执行,不是"类 Python"方言
- NumPy/pandas/Matplotlib/scikit-learn 等是**编译成 Wasm 的真实包**,`import` 时由 `loadPackagesFromImports` 自动从 CDN 拉取对应 wheel
- 绘图:matplotlib 使用 AGG 后端离屏渲染 → PNG → base64 → 显示成 `<img>`(我们注入了全局 `show()` 帮你做这件事)

### 能做什么 / 适合什么

- ✅ NumPy/pandas/SciPy/sklearn 的**全部常规用法**:数值计算、数据清洗、统计、经典机器学习(回归/分类/聚类/降维/交叉验证)
- ✅ 教学演示、算法可视化、面试手写算法验证、快速试一段代码
- ✅ `micropip.install("纯Python包名")` 可装 PyPI 上的纯 Python 包

### 限制与局限

- ❌ **没有 PyTorch / TensorFlow**(未被移植到 Pyodide;深度学习训练不在此做,推理请用 AI 工具箱)
- ❌ 不能访问本机文件系统/网络套接字(浏览器沙箱;`requests` 不可用,可用 `pyodide.http`)
- ⚠️ 性能约为本机 Python 的 1/3–1/2(Wasm 开销),大数据集(百万行以上)会吃力;内存受浏览器标签页上限(约 2–4GB)约束
- ⚠️ 含 C 扩展的包只有 Pyodide 官方移植过的才能用(NumPy/pandas/sklearn/SciPy/matplotlib 等主流都有)
- ⚠️ 运行时不持久:刷新后变量丢失(代码自己保存;需要持久 Notebook 用 JupyterLite)

## 二、AI 工具箱

### 怎么用(流程)

启动台 → **AI 工具箱**。右上角显示当前计算后端(**WebGPU ⚡** 或 Wasm)。五个标签,各自独立:

1. **情感分析**:输入英文句子 → 「分析」→ 正/负面概率条(DistilBERT,首次下载 ~65MB,底部有进度条)
2. **翻译**:输入中文 → 「翻译」→ 英文译文(OPUS-MT zh→en,~80MB)
3. **句子相似度**:第一行输入查询句,文本框每行一个候选句 → 「计算相似度」→ 按余弦相似度排序的条形图。**支持跨语言**(试试中文句和英文句比较)——这就是向量检索/RAG 的核心原理
4. **图像分类**:选本地图片 → 「识别」→ ImageNet Top-5 类别(ViT,~90MB)
5. **语音识别**:点「● 开始录音」说话 → 「■ 停止」自动转文字;或直接选音频文件(Whisper-tiny,~40MB)。首次需允许麦克风权限

### 原理

- **transformers.js**:Hugging Face Transformers 的 JS 移植,模型为 **ONNX 格式**,由 **ONNX Runtime Web** 执行
- 计算后端:**WebGPU 优先**(用你的 GPU 跑矩阵乘法,Apple Silicon 上经 Metal),个别模型不兼容时自动回退 **Wasm(CPU)**
- 模型从 Hugging Face Hub 下载,浏览器 Cache API 缓存——**下载一次,之后离线可用**
- 所有推理 100% 在本地:你的文本/图片/语音**不会上传到任何服务器**(隐私友好,这是浏览器端 AI 的核心卖点)
- 语音识别即 Whisper:与 whisper.cpp 同一模型家族,这里走 ONNX 而非 GGML,场景等价

### 能做什么 / 适合什么

- ✅ 直观体验 5 类经典深度学习任务,理解"预训练模型推理"是怎么回事
- ✅ 理解 Embedding/向量相似度(检索、去重、RAG 的基础)
- ✅ 离线/隐私场景的轻量 AI 能力原型
- ✅ 对比 WebGPU 与 Wasm 的速度差异(可在浏览器开发者工具里禁用 WebGPU 对比)

### 限制与局限

- ❌ **只有推理,没有训练/微调**(浏览器端训练不现实)
- ⚠️ 模型为小型版(tiny/base 级):Whisper-tiny 中文识别准确率一般,长句/嘈杂环境差;情感分析只支持英文;翻译质量低于商用服务——**这是模型尺寸的代价,不是 bug**
- ⚠️ 首次下载 25–90MB/模型,网络差时体验差;模型缓存占浏览器存储(可在浏览器设置里清)
- ⚠️ WebGPU 需较新浏览器(Chrome/Edge 113+);Safari 支持有限,会回退 Wasm(更慢)
- ⚠️ 大模型(LLM 对话级)不在此应用范围——7B 级模型需要 4GB+ 显存与更长加载,属于另一档工程

## 三、JupyterLite

### 怎么用(流程)

1. 启动台 → **JupyterLite**,等 JupyterLab 界面加载(首次 20–60s)
2. 双击左侧文件树的 `.ipynb` 或 File → New → Notebook,内核选 **Python (Pyodide)**
3. 像正常 Jupyter 一样写代码、Shift+Enter 逐格运行、插入 Markdown 笔记
4. Notebook **自动保存在浏览器本地存储**,下次打开还在;窗口顶栏有「新窗口打开 ↗」可全屏使用

### 原理

- JupyterLite = 完整 JupyterLab 前端 + **Pyodide 内核**,整站纯静态,无任何后端服务器
- 内核能力与「Python 实验室」同源(同一个 Pyodide),区别是 Notebook 交互形态 + 持久化
- 本应用嵌入的是 JupyterLite 官方演示站(jupyterlite.github.io)

### 适合什么 / 限制

- ✅ 结构化学习笔记:代码+图+Markdown 混排,做 ML 学习笔记本最合适
- ✅ 多单元格迭代实验(比单文件编辑器适合探索)
- ⚠️ 限制与 Python 实验室相同(无 PyTorch、性能、内存)
- ⚠️ Notebook 存在**浏览器 origin 存储**里:换浏览器/清站点数据会丢,重要内容请 File → Download 导出 `.ipynb`
- ⚠️ 嵌入的是官方演示站,其可用性依赖该站点;不可用时点「新窗口打开」或自建 jupyterlite 静态站替换 iframe 地址

## 四、复古游戏厅

### 怎么用(流程)

1. 启动台 → **复古游戏厅**
2. 选主机:NES/SNES/GB/GBA/MD/街机(MAME)/N64/PS
3. 「载入 ROM 开玩」选择**你自己合法持有**的 ROM 文件(.nes/.sfc/.gb/.gba/.md/.zip)
4. 进入模拟器:虚拟手柄/键盘可玩,EmulatorJS 自带菜单支持即时存档、读档、全屏、按键设置(界面已设为中文)

### 原理

- **EmulatorJS** 把 RetroArch 的各主机模拟核心(libretro)编译成 WebAssembly,选定主机后按需从 CDN 下载对应核心
- ROM 通过浏览器 File API 以 blob 加载,**不会上传**;存档存浏览器本地

### 适合什么 / 限制

- ✅ 课余放松;也是一个"复杂 C/C++ 项目 → Wasm"的典型工程示例(与 v86 虚拟机同类)
- ⚠️ **项目不附带任何 ROM**:请使用自己合法拥有的备份或公有领域/homebrew 作品——版权责任在使用者
- ⚠️ N64/PS 等 3D 主机对性能要求高,低配设备可能掉帧;核心需联网下载
- ⚠️ 部分街机 ROM 需要配套 BIOS 文件,缺失会启动失败(MAME 的固有要求)

## 五、整体架构与边界(为什么这样设计)

```
┌─ 浏览器(全部本地执行,无后端)────────────────────────────┐
│                                                          │
│  Python 实验室 ─┐                                        │
│  JupyterLite  ─┴─ Pyodide(CPython→Wasm)── CPU            │
│                                                          │
│  AI 工具箱 ──── transformers.js(ONNX Runtime Web)        │
│                   ├─ WebGPU(GPU,优先)                  │
│                   └─ Wasm(CPU,回退)                    │
│                                                          │
│  复古游戏厅 ─── EmulatorJS(libretro 核心→Wasm)── CPU     │
│                                                          │
│  CDN(jsdelivr / HF Hub / emulatorjs.org):              │
│    运行时 + wheel + 模型 + 模拟核心,按需下载,浏览器缓存 │
└──────────────────────────────────────────────────────────┘
```

设计取舍一览:

| 决策 | 理由 |
|---|---|
| 走 CDN 而非本地打包 | Pyodide 全家桶 + 模型动辄几百 MB,入库不现实;CDN+浏览器缓存 = 首次慢、之后快 |
| 每个应用独立 iframe | 重型 Wasm 运行时彼此隔离,崩了不连累桌面;关窗口即释放内存 |
| 推理用 transformers.js 而非 whisper.cpp 原版 | 同等场景(Whisper 语音识别)下免去自行编译 wasm + 托管模型,且统一获得 WebGPU 加速与模型缓存 |
| 启动台启动、非常驻 Dock | 学习工具按需使用,不挤占日常 Dock |

一句话总结边界:**这套工具覆盖"经典 ML 全流程学习 + 深度学习推理体验",不覆盖"深度学习训练与大模型"**——后者请在本机装 PyTorch/conda,或用云端 GPU。

## 六、常见问题

**Q:打开应用一直在转圈?**
A:都依赖 CDN(jsdelivr/HF Hub/emulatorjs.org),检查网络;首次加载本来就慢(运行时 10MB、模型最大 90MB),UI 有进度提示。

**Q:模型下载到哪了?会重复下载吗?**
A:浏览器 Cache API/HTTP 缓存,同一浏览器不会重复下载;清浏览器站点数据会清掉,下次重新下载。

**Q:为什么 Whisper 识别我说的中文不太准?**
A:用的是 tiny 档模型(40MB),为加载速度牺牲精度。安静环境、吐字清晰、句子短会明显改善。

**Q:能跑 ChatGPT 那样的对话大模型吗?**
A:本套件未包含。技术上 transformers.js/WebLLM 可以跑 1–8B 量化模型,但需 4GB+ 显存和数 GB 下载,体验定位不同,故未纳入。

**Q:Python 实验室里能 pip install 吗?**
A:用 `import micropip; await micropip.install("包名")`(纯 Python 包);含 C 扩展的包仅限 Pyodide 官方已移植列表。
