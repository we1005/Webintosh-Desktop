# Webintosh 启动与使用指南

本文档说明如何启动 Webintosh、各个入口的区别、内置应用的用法，以及常见问题处理。

---

## 一、单一仓库结构

Webintosh 现已合并为**单一自包含仓库** `Webintosh-Desktop/`，不再依赖任何外部站点：

| 部分 | 位置 | 说明 |
|------|------|------|
| **桌面本体** | 仓库根（`index.html`） | 真正的"系统"：菜单栏、Dock、窗口管理、文件系统、全部内置应用。 |
| **开机体验** | `boot-experience/` | 开机欢迎页 → 固件自检 → 启动 → 登录，自成一体（含自己的 `assets/`、`src/js/`）。登录成功后跳转到桌面本体。 |

> `boot-experience/` 是从原 `Webintosh/` 入口站整体迁入的，路径自包含，与桌面侧资源互不干扰。

完整链路：`boot-experience/`（开机欢迎 → firmware → boot → 登录）→ 跳转 → 仓库根（桌面）。

## 二、启动方法

整个项目是**纯静态站点**（HTML/CSS/JS），无需构建、无需安装依赖即可运行。只需要一个静态文件服务器，且必须通过 HTTP 访问（不能用 `file://` 直接打开，因为用到了 ES Module 与 `fetch`）。

### 方式 A：纯前端（推荐，零依赖）

以 `Webintosh-Desktop` 的父目录为 Web 根（或直接以 `Webintosh-Desktop` 为根，见下方注）：

```bash
cd /path/to/personal          # 该目录下含 Webintosh-Desktop/
python3 -m http.server 8765
```

然后浏览器打开任一入口：

| 入口 | 地址 |
|------|------|
| **桌面（直接进，跳过登录）** | http://localhost:8765/Webintosh-Desktop/?desktop |
| **完整开机流程** | http://localhost:8765/Webintosh-Desktop/boot-experience/ |
| **iPhone 5s 模式** | http://localhost:8765/Webintosh-Desktop/ios/ |

> 完整流程的登录密码任选其一：`Ventura`、`Sonoma`、`Sequoia`。
>
> **注**：也可直接以 `Webintosh-Desktop/` 为 Web 根启动（`cd Webintosh-Desktop && python3 -m http.server 8765`），此时入口去掉路径中的 `Webintosh-Desktop/` 前缀即可（如 `http://localhost:8765/boot-experience/`）。

此方式下，文件系统使用浏览器原生的 **OPFS**（Origin Private File System）——数据持久保存在浏览器里，刷新、重启浏览器都不会丢失。**无需后端**。

### 方式 B：带 FastAPI 后端（可选，多端共享文件）

只有当你想把文件系统切换为"服务器存储"（多设备共享、真实账户场景）时才需要。

首次准备虚拟环境（用 `uv`，因为部分系统自带 Python 的 pip 不可用）：

```bash
cd /path/to/personal/Webintosh-Desktop/server
uv venv --python 3.12 .venv
uv pip install --python .venv/bin/python fastapi uvicorn
```

启动：

```bash
cd /path/to/personal/Webintosh-Desktop/server
./.venv/bin/python main.py        # 默认监听 0.0.0.0:8787
```

FastAPI 服务器同时**静态托管整个站点**，所以也可以只跑这一个进程：

```
http://localhost:8787/Webintosh-Desktop/?backend=remote
```

数据存放在 `server/data/`（已在 `.gitignore` 中，不进版本库）。

## 三、切换文件系统后端

前端默认用 OPFS。切换到 FastAPI（`remote`）有三种方式，任选：

1. **URL 参数**：在地址后加 `?backend=remote`
2. **终端命令**：在内置"终端"里输入 `backend remote`（切回 `backend opfs`）
3. **localStorage**：设置 `webintosh.backend` 为 `remote`

> 切到 `remote` 时若服务器不可达，会自动回退到 OPFS，并在终端里红字提示。

## 三-B、配置开机主题

开机画面（固件自检 / 启动）有两种背景，默认**黑色**。改 `boot-experience/config.js` 一处即可：

```js
window.BOOT_CONFIG = {
    // "black"（默认）| "gray" | "random"
    theme: "black",
};
```

- `"black"` —— 黑色开机背景（Apple Silicon / 新机型风格）
- `"gray"` —— 浅灰开机背景（部分 Intel Mac 风格）
- `"random"` —— 每次开机随机二选一

也可**不改文件、临时覆盖**：在开机入口加 URL 参数 `?theme=gray`（优先级高于配置），例如
`http://localhost:8765/boot-experience/?theme=gray`。主题会一路透传给 firmware → boot，全流程配色一致。

## 四、内置应用用法

从 **Dock** 点击图标启动，或在**终端**里用 `open <应用名>` 启动。

### 访达（Finder）
- 真实文件管理：浏览 / 新建文件夹 / 新建文本文件 / 新建 Markdown 文档 / 重命名 / 删除（移到废纸篓）
- 侧边栏：桌面、文稿、下载、音乐、图片、应用程序
- **Sequoia 风格工具栏**：返回 / 前进、**显示方式分段控件（图标 / 列表 / 分栏 / 画廊）**、排序方式（名称 / 种类）、共享（拷贝路径）、**右上角搜索框**（当前目录实时按名称过滤，状态栏显示「命中 / 总数」）
- 底部**路径栏**显示完整层级（Macintosh HD › … ），可点击跳转
- **重命名为 macOS 行内蓝条**：双击右键「重命名」就地编辑，默认仅选中基名、扩展名固定为灰色后缀不被误改，回车提交 / Esc 取消；非法字符与重名会被拦截
- 双击文件夹进入；双击文本文件（`.txt/.js/.json/.css/.html` 等）→「文本编辑」；`.md` →「Typora」
- 任意应用对文件的改动会让访达**自动刷新**（跨窗口同步）

### 终端（Terminal）
- 外观对标 macOS Terminal.app（深色半透明、SF Mono 字体）
- 命令：`help` `ls [-l]` `cd` `pwd` `cat` `echo（支持 > 与 >> 重定向写文件）` `mkdir` `touch` `rm [-r]` `mv` `cp [-r]` `clear` `date` `whoami` `uname [-a]` `history` `df` `backend [opfs|remote]` `open <应用名>` `neofetch`
- 支持：Tab 路径补全、↑↓ 命令历史、Ctrl+C 清行、Ctrl+L 清屏
- 试试 `neofetch`（彩虹 ASCII 苹果 logo + 系统信息）

### 文本编辑（TextEdit）
- 打开 / 编辑 / 保存文本文件到文件系统
- `Cmd+S`（或 `Ctrl+S`）保存；有未保存修改时标题显示「— 已编辑」
- 与访达、终端共享同一份文件

### Typora —— Markdown 编辑器（启动台）
- 基于 [Vditor](https://github.com/Vanessa219/vditor)，**Typora 风格即时渲染**（也可切换所见即所得 / 分屏预览）
- 表格、数学公式、Mermaid、代码高亮、任务列表全支持
- 与文件系统打通：`⌘S` 保存到 /Documents，「打开…」列出全部 .md；**访达/桌面双击任何 .md 都用它打开**（其余文本文件仍走文本编辑）；桌面与访达空白处右键均有「新建 Markdown 文档」
- 「导出 .md」可下载到本机；需联网加载 CDN

### 虚拟机（基于 v86）
- 窗口内**真实启动 FreeDOS**（x86 模拟器，运行在 WebAssembly）
- 启动后可在黑屏里输入 DOS 命令：`dir`、`ver`，或玩 `invaders` / `tetris` / `snake`
- 工具条：暂停 / 继续、重启、Ctrl+Alt+Del、载入自定义镜像（`.img` 当软盘 / `.iso` 当光盘）
- 窗口不可见时自动暂停 CPU，省电

### 🎵 Spotify
- 暗色界面复刻：资料库、悬停播放钮、播放行均衡器、底部播放条
- 右下角按钮进**全屏频谱可视化**（WebGPU 渲染，自动回退 WebGL2）

### 🍎 音乐（Apple Music）
- 浅色界面复刻：顶部 LCD 播放器、侧边栏
- 点 LCD 进**沉浸式播放页**：流体氛围背景（WebGPU）、弹簧封面、逐行歌词（点击歌词可跳转）

> 两个播放器的曲目均为运行时用 `OfflineAudioContext` 合成，封面由 Canvas 程序化生成，**无任何外部音视频/图片资源**。

### 机器学习套件（启动台）
从「启动台」启动，**需联网**（库与模型走 CDN，按需下载并被浏览器缓存）。
**详细的用法 / 原理 / 限制 / 适用场景见专篇：[机器学习套件](机器学习套件.md)**。速览：
- **Python 实验室**（Pyodide）：左侧写 Python 代码，⌘↵ 运行；顶部下拉有 ML 示例（线性回归/KMeans/决策边界/pandas/梯度下降）；matplotlib 图直接显示，`import` 的包自动安装。
- **AI 工具箱**（transformers.js，WebGPU 优先）：五个标签——情感分析、中英翻译、句子相似度、图像分类、Whisper 语音识别（点录音或选音频文件）。首次使用各功能会下载 25–90MB 模型。
- **JupyterLite**：完整 Jupyter Notebook，零后端，笔记本保存在浏览器。
- **复古游戏厅**（EmulatorJS）：选主机核心 → 载入你自己的 ROM 文件（.nes/.gba/.zip 等）。

### 桌面图标
- `/Desktop` 目录里的文件会实时渲染为桌面图标，双击打开。

## 六、iPhone 5s 模式（iOS 7）

- **触屏小屏设备**访问 `Webintosh/` 或 `Webintosh-Desktop/` 会**自动重定向**到 `ios/`
- **桌面浏览器**想看这个模式：直接访问 `…/Webintosh-Desktop/ios/`，会在一个 CSS 绘制的 iPhone 5s 机身边框里显示
- 想在桌面浏览器**强制进桌面版**（不跳 iOS）：地址加 `?desktop`
- 操作：横向滑动解锁 → SpringBoard；点「音乐 / Spotify」全屏打开，底部 home 横条返回；其余图标弹 iOS 7 风格对话框

## 七、常见问题

**Q：改了 HTML/CSS/JS，刷新后没变化？**
A：不用重启服务器（静态服务器永远读最新文件）。做一次**硬刷新** `Cmd+Shift+R` 即可。应用窗口的资源加载已带版本戳（`?v=时间戳`），之后普通刷新或重开窗口都会取最新文件。

**Q：需要重启服务器吗？**
A：只有改了 `server/main.py`（FastAPI 逻辑）或服务器进程被关时才需要。纯前端改动只刷浏览器。

**Q：`file://` 直接打开不行？**
A：对，必须通过 HTTP（ES Module / fetch / OPFS 都要求安全上下文）。用上面的静态服务器即可。

**Q：数据存在哪里？会丢吗？**
A：OPFS 模式存在浏览器的源私有存储里，刷新/重启浏览器不丢（清除浏览器站点数据会清空）。remote 模式存在 `server/data/`。

**Q：虚拟机黑屏 / 启动失败？**
A：v86 运行时和 FreeDOS 镜像已本地化在 `apps/vm/`，无需联网。若失败，窗口内有"重试"按钮。
