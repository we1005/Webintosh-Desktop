# Webintosh 架构说明

本文档说明 Webintosh-Desktop 的目录结构、核心机制（文件系统、窗口、应用加载、GPU 渲染、iOS 模式），供开发者理解与扩展。

---

## 一、目录结构

```
Webintosh-Desktop/
├── index.html                     # 桌面入口（加载下列脚本/样式 + 触屏重定向）
├── boot-experience/               # 开机体验（自包含：从原 Webintosh 入口站迁入）
│   ├── index.html                 # OOBE 欢迎页 → firmware
│   ├── firmware/                  # 固件自检 + manager
│   ├── boot/                      # 启动动画
│   ├── logon/                     # 登录页（密码正确 → 跳转 ../../ 即桌面根）
│   ├── recovery/                  # 恢复模式
│   ├── assets/                    # 本流程专用样式/图片/声音/字体（与桌面侧隔离）
│   └── src/js/                    # 本流程脚本（boot/firmware/logon/recovery…）
├── os/                            # WebOS 核心
│   ├── vfs.js                     # 虚拟文件系统（OPFS / FastAPI 双后端门面）
│   └── desktop-sync.js            # 把 /Desktop 渲染为桌面图标
├── apps/                          # iframe 型应用 + 共享资源
│   ├── music-core.js              # 音乐引擎：合成曲目 + 程序化封面 + WebGPU/WebGL 渲染面
│   ├── fonts.css                  # 桥接字体族名到 assets/fonts
│   ├── spotify/index.html         # Spotify 播放器（完整页面，跑在 iframe 内）
│   ├── applemusic/index.html      # Apple Music 播放器（完整页面，跑在 iframe 内）
│   └── vm/                        # v86 虚拟机
│       ├── index.html             # iframe 宿主页
│       ├── libv86.js              # v86 运行时
│       ├── v86.wasm               # x86 模拟器 (WebAssembly)
│       ├── seabios.bin            # SeaBIOS（取自 v86 master，与 libv86 版本配对）
│       ├── vgabios.bin            # VGA BIOS（同上）
│       └── images/freedos722.img  # FreeDOS 可启动软盘镜像
├── ios/                           # iPhone 5s（iOS 7）移动端模式
│   ├── index.html
│   ├── ios.css
│   └── ios.js
├── server/                        # 可选 FastAPI 后端
│   ├── main.py                    # 文件系统 REST API + 全站静态托管
│   ├── requirements.txt
│   ├── .venv/                     # 虚拟环境（gitignored）
│   └── data/                      # remote 后端数据（gitignored）
├── assets/
│   ├── apps/<应用名>.html          # 桌面应用的 HTML 片段（fragment）
│   ├── stylesheets/
│   │   ├── ui/ui.css              # 窗口基类（position/overflow/圆角 + 交互态）★集中兜底
│   │   ├── finderbar/             # 菜单栏
│   │   ├── dock/                  # Dock
│   │   ├── main/                  # 桌面/壁纸
│   │   └── apps/<应用名>/index.css # 各应用样式
│   ├── icons/                     # Dock/应用图标（svg）
│   ├── images/                    # 壁纸、文件夹/文档图标等
│   └── fonts/                     # SF Pro / PingFang / SF Mono
└── src/javascripts/
    ├── main.js                    # 桌面初始化
    ├── finderbar.js               # 菜单栏（Apple 菜单、应用菜单、时钟、状态图标）
    ├── dock.js                    # Dock（放大动画、运行指示、启动应用）
    ├── desktop.js                 # 桌面（右键菜单、图标、拖放）
    ├── window.js                  # 窗口管理器（创建/拖拽/缩放/红绿灯/最小化）
    ├── apps/<应用名>.js            # 各应用逻辑
    └── ui/                        # contextMenu / alert / menu / notification
```

## 二、应用加载机制（fragment 模式）

桌面应用 = **三个同名文件**：

1. `assets/apps/<名>.html` —— HTML **片段**（非完整页面），根元素 `<div class="<类名> window">`，含 `.topbar > .wintools`（红黄绿三灯）。
2. `assets/stylesheets/apps/<名>/index.css` —— 该应用样式，选择器以应用类名前缀避免污染。
3. `src/javascripts/apps/<名>.js` —— ES Module，应用逻辑。

`window.js` 的 `create(file, name, light, centered)` 负责：
- `fetch` 片段 HTML 注入主文档 `<body>`
- 动态插入对应的 `<script type="module">` 和 `<link>`（三者都带 `?v=时间戳` 缓存破坏）
- 绑定红绿灯、标题栏拖拽、缩放手柄、焦点置顶

> 注意：应用 JS 运行在**主文档**（不是 iframe）。重复打开会再次注入执行，代码需用 `dataset` / `window` 级守卫防重复绑定。
> 音乐 / Spotify / 虚拟机是**例外**：它们的片段里嵌 `<iframe>`，真正的应用是 `apps/<x>/index.html` 完整页面，跑在 iframe 沙箱内。

### 窗口基类集中在 ui.css（重要约定）

历史上 `assets/stylesheets/ui/window.css` 是**从未被加载的死文件**。窗口的通用样式统一由 `ui/ui.css` 的 `.window` 规则兜底：

```css
.window {
  position: absolute;   /* body 是 flex(space-between)，非 absolute 会挤动 Dock 且无法拖动 */
  overflow: hidden;     /* 裁切内容到圆角，避免 iframe 方角溢出 */
  border-radius: 11px;  /* 各应用可覆盖 */
}
```

新增窗口**不要**假设有别的基类来源；通性都加在这里。

## 三、文件系统（VFS）

`os/vfs.js` 是统一门面，对上提供 async API，对下切换两种后端：

- **OPFS 后端（默认）**：浏览器 Origin Private File System，纯前端持久化。
- **RemoteBackend**：对接 `server/main.py` 的 REST API（`/api/fs/list|read|write|mkdir|rm|mv|stat`）。

切换优先级：URL `?backend=` > `localStorage.webintosh.backend` > 默认 `opfs`；remote 不可达自动回退 opfs。

**跨窗口同步**：所有写操作经 `BroadcastChannel('webintosh-vfs')` 广播 `change` 事件，访达 / 终端 / 文本编辑 / 桌面图标据此自动刷新。

主要 API：
```js
import vfs, { normPath, dirname, basename, joinPath } from "os/vfs.js";
await vfs.list(path)        // [{name, kind:'file'|'dir', size, mtime}]
await vfs.readText(path) / readBytes(path)
await vfs.writeText(path, str) / writeBytes(path, bytes)
await vfs.mkdir(path) / rm(path) / mv(src,dst) / exists(path)
await vfs.setBackend('opfs'|'remote')
vfs.backendName
vfs.addEventListener('change', e => e.detail.path)
```

默认会种入目录 `/Desktop /Documents /Downloads /Music /Pictures /Applications` 和一个欢迎文件。

## 四、GPU 渲染

### 合成器路径动画
Dock 放大、窗口拖拽只触碰 `transform`（`scale` / `translate3d`），由 GPU 合成器驱动，主线程零布局抖动：
- Dock：图标 `scale` + 容器 `translate3d` 推挤 + 玻璃背景 `scaleX` 跟随展宽（不再逐帧改 `width/height`）
- 窗口拖拽：拖动期间只写 `translate3d`，松手一次性提交 `left/top`；交互期间 `body.wm-interacting` 屏蔽 iframe 指针事件防丢帧

### WebGPU 优先、WebGL2 回退
`apps/music-core.js` 的 `createSurface(canvas, { wgsl, glsl })`：
- 有 `navigator.gpu` 时用 **WebGPU（WGSL 着色器）**，经 Dawn → Metal
- 否则回退 **WebGL2（GLSL）**
- 两套着色器同语义（uniforms：res/time/level/调色板，频谱纹理）
- 用于 Spotify 全屏频谱、Apple Music 沉浸式流体背景

### 音频合成
8 首曲目（2 张专辑、4 种曲风）由 `OfflineAudioContext` 在独立线程离线合成（鼓组/贝斯/合成垫/琶音/卷积混响），播放时经 `AnalyserNode` 实时 FFT 供可视化采样。专辑封面由 Canvas 程序化生成。**零外部音视频/图片资源**。

## 五、iOS 模式

`ios/` 是自成一体的页面（独立于桌面）。触屏小屏设备访问桌面入口时，`index.html` 顶部脚本检测 `pointer:coarse` + 屏幕尺寸自动 `location.replace('./ios/')`（`?desktop` 可绕过）。

- 锁屏：超细字重时钟、滑动解锁手势（拖拽 + 回弹 / 完成）
- SpringBoard：CSS/SVG 手绘 iOS 7 扁平图标（日历显示真实日期、时钟真实走针）、毛玻璃 Dock、视差壁纸
- 音乐 / Spotify 全屏 iframe 复用桌面的播放器页面
- 桌面浏览器访问时，用纯 CSS 绘制 iPhone 5s 机身边框包裹 320×568 屏幕

## 六、测试方法（交互验证）

UI 改动通过 CDP（Chrome DevTools Protocol）驱动 headless Edge 做**真实鼠标/键盘事件**逐项断言，并整图截图核对全局布局。已建立 5 个交互测试矩阵：

| 矩阵 | 覆盖 |
|------|------|
| A | 窗口管理：每类窗口 × {拖动/红绿灯/全屏/拉伸/最小化/缩放/置顶} |
| B | 菜单栏 + Dock + 桌面右键 + 启动台 |
| C | 访达 + 终端 + 文本编辑 全交互 + 跨应用同步 |
| D | 虚拟机（FreeDOS 启动/键盘/按钮）+ 两个音乐播放器控件 |
| E | iOS 模式全手势 + 整图布局核对 |

### 测试注意（headless 的坑）
- **缓存**：测试用全新 `user-data-dir` 无缓存，会掩盖真实用户的浏览器缓存问题；前端资源已加 `?v=` 版本戳应对。
- **命中测试**：headless 下对动态上下文菜单 `<p>` 的合成点击可能落空，菜单项激活可用元素 `.click()`；其余交互用真实 Input 事件（点击前先 `mouseMoved`）。
- **触摸模拟复位**：测完移动视口后要关闭 `setTouchEmulationEnabled`，否则 `pointer:coarse` 影响桌面态断言。
- **v86 键盘**：headless 焦点抓取不可靠，键盘输入需在**有头**浏览器验证。
- **进程清理**：测试启动的 Edge 用独立 `--user-data-dir=/tmp/edge-<名>`，清理时只按该唯一标识定点，**不可**用宽匹配 `pkill node` / `pkill "Microsoft Edge"`（会误杀用户进程）。

## 七、已知与真 macOS 的差距

以下为已识别、尚未修复的还原度差距（按影响排序）：

1. 菜单栏"文件/编辑/显示/前往/窗口/帮助"暂无下拉内容（仅  和访达菜单可展开）
2. 访达工具栏：面包屑应移到底部路径栏；缺搜索框与视图切换段控
3. 红绿灯悬停未显示 `✕ − +` 符号
4. "关于本机"窗口偏透明（真机接近不透明浅灰）
5. 菜单栏时钟用了全角冒号、日期格式与 macOS 略有出入
6. 最小化是"缩放飞入"而非默认"神灯"动画

详见 README「路线图 / 已知差距」。
